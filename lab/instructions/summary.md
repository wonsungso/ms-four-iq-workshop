# 요약

Azure AI Search와 다양한 소스 유형으로 에이전트형 지식 베이스를 구축하는 랩을 완료했습니다. 5개의 단계별 노트북을 통해 여러 문서를 기반으로 한 멀티 소스 지식 베이스를 구축하고, MCP를 통해 웹 검색 결과를 추가하고, Fabric IQ와 Work IQ를 추가하고, Work IQ와 Fabric IQ를 결합했습니다.

## 학습 계속하기

- [Azure AI Search](https://learn.microsoft.com/azure/search/) - Azure AI Search의 전체 기능과 다른 Azure 서비스와의 통합 방식을 살펴봅니다.
- [Azure AI Search에서 에이전트형 검색을 위한 인덱스 설계](https://learn.microsoft.com/azure/search/search-agentic-retrieval-how-to-index) - 에이전트형 검색을 위한 데이터 구조화 모범 사례.
- [Azure AI Search에서 지식 베이스 만들기](https://learn.microsoft.com/azure/search/search-agentic-retrieval-how-to-create?tabs=rbac%2Cpython-get-agents%2Cpython-create-agent%2Cpython-query-agent%2Cpython-delete-agent) - 지식 베이스 생성 및 구성 단계별 가이드.
- [지식 소스 만들기](https://learn.microsoft.com/azure/search/search-knowledge-source-overview) - 다양한 데이터 소스를 생성하고 구성하는 방법을 이해합니다.
- [Azure AI Search에서 인용 기반 응답을 위한 답변 합성 사용](https://learn.microsoft.com/azure/search/search-agentic-retrieval-how-to-synthesize) - 답변 합성을 사용해 인용이 포함된 근거 기반 답변을 생성하는 방법을 배웁니다.


## 다음 단계

배운 내용을 적용할 수 있는 몇 가지 방법입니다.

**더 많은 MCP 소스 추가** - 추가 MCP 서버를 연결하여 지식 베이스가 실시간으로 질의할 수 있는 범위를 넓힙니다.

**검색 동작 사용자 지정** - 추론 노력 수준을 조정하고, 특정 소스에 대한 사용자 지정 검색 및 답변 지침을 구성합니다.

**내 데이터와 통합** - 샘플 HR 및 건강 보험 인덱스를 본인의 엔터프라이즈 콘텐츠로 교체합니다.

**대화형 인터페이스 구축** - 추가 도구와 채팅 UI로 Foundry 에이전트를 확장하여 최종 사용자에게 제공합니다.
