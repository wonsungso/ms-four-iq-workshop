# AI 에이전트 가이드라인

이 파일에는 이 저장소에서 작업하는 AI 에이전트를 위한 지침과 가이드라인이 들어 있습니다.

## 🔒 보안 모범 사례

**민감한 정보를 이 저장소에 절대 커밋하지 마세요.**
- API 키, 토큰, 자격 증명
- 개인용 액세스 토큰(PAT)
- 암호가 포함된 데이터베이스 연결 문자열
- 환경별 구성 값

**MCP 구성 파일(`mcp.json`)의 경우:**
- `"YOUR_API_KEY_HERE"` 또는 `"${API_KEY}"` 같은 자리 표시자 값을 사용하세요
- 민감한 데이터는 환경 변수를 참조하세요
- 필요한 환경 변수에 대한 문서를 포함하세요

## 📋 저장소 가이드라인

### 목적
이 저장소는 Microsoft Build 2026 세션 콘텐츠 저장소이며 다음을 충족해야 합니다.
- 세션 참가자에게 명확하고 실행 가능한 콘텐츠 제공
- 원격/자가 학습자를 위한 자기 주도 학습 지원
- GUIDANCE.md에서 정한 구조 준수

### 허가 없이 수정하면 안 되는 항목:
- 라이선스 파일(`LICENSE`, `LICENSE-DOCS`, `CODE_OF_CONDUCT.md`)
- 보안 파일(`SECURITY.md`)
- `.github/` 디렉터리의 GitHub 워크플로 파일

### 콘텐츠 규칙
- 저장소에 대용량 바이너리 파일(PowerPoint 덱, 동영상, 녹화본)을 두지 않음
- 슬라이드와 녹화본 링크는 괜찮음 - 실제 파일만 호스팅하지 않으면 됨
- 모든 README 파일은 최신 상태로 유지해야 함
- 사용되지 않는 폴더(자리 표시자 README만 포함된 폴더)는 릴리스 전에 제거해야 함
- `.ipynb` 파일을 커밋하기 전에 항상 노트북 출력을 지우고 실행 횟수를 초기화해야 함

### 이슈 관리
사용자가 문제를 보고하거나, 추적해야 할 질문을 하거나, 이슈를 제기하려는 경우:

1. **사용 가능한 템플릿 확인** - `.github/ISSUE_TEMPLATE/`에서 `.yml` 또는 `.md` 템플릿 파일이 있는지 확인합니다. 각 템플릿이 기대하는 필드와 레이블을 이해하기 위해 읽어 봅니다.
2. **요청을 템플릿에 매칭** - 사용자가 설명하는 내용을 바탕으로 가장 적합한 템플릿을 고릅니다. 템플릿이 없으면 일반 이슈를 생성합니다.
3. **사용자가 필드를 채우도록 도움** - 템플릿의 필수 필드를 대화식으로 살펴보며 가능한 경우 답변을 제안합니다.
4. **이슈 생성** - 템플릿이 일치하면 `gh issue create --template <template-file>` 을, 일반 이슈는 `gh issue create` 를 사용합니다.
5. **레이블 적용** - `gh label list` 로 저장소에 존재하는 레이블을 확인합니다. 이슈 유형에 따라 관련 레이블을 적용합니다. 존재하지 않는 레이블은 적용하려 하지 마세요.

각 단계가 시작될 때 열린 이슈를 검토할 때는 요약하고 조치를 제안하세요 - 이 동작은 이미 GUIDANCE.md의 이슈 추적 및 커밋 섹션에 존재합니다.

### 시작하기
이 저장소에 아직 `GUIDANCE.md` 파일이 있다면, 이는 설정이 아직 완료되지 않았다는 뜻입니다. 해당 파일을 읽고 지침을 따라 저장소를 게시 준비 상태로 만드세요.

## 📓 노트북 스타일 가이드라인

이 규칙은 `notebooks/`의 모든 `.ipynb` 파일에 적용됩니다.

### 작성 스타일
- 산문 설명에서 세미콜론을 사용하지 마세요. 대신 문장을 다시 작성하세요.
- 산문 설명에서 엠 대시(`-`)를 사용하지 마세요. 마침표로 새 문장을 시작하거나 구조를 바꾸세요.
- 일반 대시(`-`)는 목록에서만 사용하고, 실행 중인 문장에서 절 구분자로 사용하지 마세요.

### 구조 (노트북별)

- **첫 번째 셀**: 제목 + 시나리오 설정 + `**🎯 미션**` 글머리 기호 결합.
- **설정/시나리오**: 기술적 기능을 배워야 하는 이유를 동기 부여하는 Part X 내러티브("Zava의 Part 1…")로 작성.
- **미션 글머리 기호**: 참가자가 만들 내용과 일치하는 1~3개 항목. 끝의 `## ✅ 미션 완료` 셀은 동일한 성과를 다루되 더 충실한 설명으로 확장할 수 있음.
- **빌딩 블록**: 선택 사항. 포함할 경우 이 노트북에서 새롭게 등장하는 개념만 설명하고 앞 파트에서 이미 소개한 개념은 다시 설명하지 않음.
- **마지막 셀**(Continue 링크 앞): `**무엇을 만들었나:**` 와 `✓` 글머리 기호가 있는 `## ✅ 미션 완료`. 캡스톤 노트북(Part 5)은 대신 `**랩 전체에서 만든 것:**` 을 사용.
- **소스 헌트**: 각 노트북에서 참조 출력 셀과 활동 로그 셀 뒤에 `#### 🔍 소스 헌트` 마크다운 셀을 포함.
